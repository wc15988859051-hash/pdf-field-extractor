- 不要关闭浏览器窗口
- 耐心等待，大约 5-10 分钟
- 构建过程会自动下载 node:20-alpine 镜像

## 🆘 快速故障排查

### 问题：找不到 "Compose" 菜单

**可能原因**：
- NAS 的 Docker 管理界面可能名称不同

**解决方案**：
- 查找类似 "项目"、"项目组"、"堆栈" 等菜单
- 或使用"容器"菜单手动创建容器（见附录）

---

### 问题：构建失败

**可能原因**：
- 网络连接问题
- 文件未完整上传
- 存储空间不足

**解决方案**：
1. 检查网络连接
2. 确认所有文件都已上传
3. 检查 NAS 存储空间（至少需要 2GB）

---

### 问题：容器无法启动

**可能原因**：
- 端口 5000 被占用
- 配置错误

**解决方案**：
1. 检查是否有其他容器使用端口 5000
2. 查看容器日志，找到错误信息
3. 删除容器和镜像，重新创建

---

### 问题：无法访问应用

**检查清单**：
```
□ 容器是否在运行？
  → 在"容器"菜单中查看状态

□ 端口映射是否正确？
  → 本地端口:5000 → 容器端口:5000

□ 防火墙是否开放端口 5000？
  → 在 NAS 网络设置中检查

□ NAS IP 地址是否正确？
  → 在 NAS 设置中查看 IP 地址
```

## 📱 日常管理命令

### 重启服务
```
1. 进入 Docker 管理界面
2. 点击"容器"菜单
3. 找到 pdf-extractor 容器
4. 点击"..."
5. 选择"重启"
```

### 停止服务
```
1. 进入 Docker 管理界面
2. 点击"容器"菜单
3. 找到 pdf-extractor 容器
4. 点击"..."
5. 选择"停止"
```

### 查看日志
```
1. 进入 Docker 管理界面
2. 点击"容器"菜单
3. 点击 pdf-extractor 容器
4. 选择"日志"标签
5. 查看实时日志
```

### 更新应用
```
1. 上传新文件到 NAS
2. 进入"Compose"菜单
3. 找到 pdf-extractor 项目
4. 点击"..."
5. 选择"重新构建"
6. 构建完成后点击"重启"
```

## 💡 关键提示

1. **首次构建慢**：第一次需要 5-10 分钟，正常现象
2. **不要中断**：构建过程中不要关闭浏览器
3. **查看日志**：遇到问题首先查看日志
4. **备份数据**：定期备份 data 目录
5. **监控资源**：关注 CPU 和内存使用
